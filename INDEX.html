Estructuras Condicionales

Si-Entonces / SiNo

Las estructuras condicionales permiten que un algoritmo tome decisiones y ejecute diferentes bloques de código según se cumplan o no ciertas condiciones. Son la base de la lógica de programación.

Flujo de decisión: ¿Es mayor de edad?
Inicio
Edad >= 18?
Mayor de edad ✓
Menor de edad ✗
Fin
Haz clic en "Simular 25 años" o "Simular 15 años"
📋 Código PSeInt 
Proceso VerificarEdad
    Definir edad Como Entero
    
    Escribir "Ingrese su edad:"
    Leer edad
    
    Si edad >= 18 Entonces
        Escribir "Es mayor de edad"
    SiNo
        Escribir "Es menor de edad"
    FinSi
FinProceso

💡 Aplicación práctica

Sistemas de validación: verificar si un usuario puede acceder a contenido restringido, determinar si una nota es aprobatoria (>=60), o clasificar el IMC de una persona.

Tiempo O(1)

Estructuras Repetitivas

Para / Mientras / Repetir

Los ciclos o bucles permiten ejecutar un bloque de código múltiples veces. Son esenciales para procesar colecciones de datos, realizar cálculos iterativos y automatizar tareas repetitivas.

Ciclo Para: Tabla del 5
1
5 × 1 = ?
5 × 2 = ?
5 × 3 = ?
5 × 4 = ?
5 × 5 = ?
Haz clic en "Ejecutar Ciclo" para ver la iteración
📋 Código PSeInt 
Proceso TablaMultiplicar
    Definir i, resultado Como Entero
    
    Escribir "Tabla del 5:"
    
    Para i <- 1 Hasta 5 Con Paso 1 Hacer
        resultado <- 5 * i
        Escribir "5 × ", i, " = ", resultado
    FinPara
FinProceso

💡 Aplicación práctica

Generar reportes de nómina procesando empleados uno por uno, validar que todos los campos de un formulario estén completos, o calcular el factorial de un número.

Tiempo O(n)

Arreglos (Vectores)

Estructuras de Datos

Los arreglos son estructuras que almacenan múltiples valores del mismo tipo bajo un solo nombre. Permiten organizar datos relacionados y acceder a ellos mediante índices numéricos.

Ordenamiento Bubble Sort en Vector
Haz clic en "Ordenar Vector" para comenzar
📋 Código PSeInt 
Proceso OrdenarVector
    Definir nums, i, j, temp Como Entero
    Dimension nums[5]
    
    // Llenar vector
    nums[1] <- 45; nums[2] <- 12
    nums[3] <- 78; nums[4] <- 23
    nums[5] <- 67
    
    // Bubble Sort
    Para i <- 1 Hasta 4 Hacer
        Para j <- 1 Hasta 5-i Hacer
            Si nums[j] > nums[j+1] Entonces
                temp <- nums[j]
                nums[j] <- nums[j+1]
                nums[j+1] <- temp
            FinSi
        FinPara
    FinPara
FinProceso

Búsqueda Binaria

Divide y Vencerás

Algoritmo eficiente para encontrar elementos en arreglos ordenados. Divide repetidamente el espacio de búsqueda a la mitad, descartando la mitad donde no puede estar el objetivo.

Buscando el número 42
10
25
32
42
58
67
89
95
Haz clic en "Buscar" para iniciar la búsqueda binaria
📋 Código PSeInt 
Proceso BusquedaBinaria
    Definir nums, buscado, izq, der, medio Como Entero
    Dimension nums[8]
    Definir encontrado Como Logico
    
    // Vector ordenado
    nums[1]<-10; nums[2]<-25; nums[3]<-32; nums[4]<-42
    nums[5]<-58; nums[6]<-67; nums[7]<-89; nums[8]<-95
    
    buscado <- 42
    izq <- 1; der <- 8; encontrado <- Falso
    
    Mientras izq <= der Y encontrado = Falso Hacer
        medio <- trunc((izq + der) / 2)
        Si nums[medio] = buscado Entonces
            encontrado <- Verdadero
        SiNo
            Si buscado < nums[medio] Entonces
                der <- medio - 1
            SiNo
                izq <- medio + 1
            FinSi
        FinSi
    FinMientras
FinProceso

Funciones y Recursividad

Modularidad

Las funciones permiten dividir un programa en módulos reutilizables. La recursividad ocurre cuando una función se llama a sí misma para resolver subproblemas más pequeños hasta alcanzar un caso base.

Pila de llamadas: Factorial(5)
factorial(5) = 5 × factorial(4)
factorial(4) = 4 × factorial(3)
factorial(3) = 3 × factorial(2)
factorial(2) = 2 × factorial(1)
factorial(1) = 1 (Caso Base)
Haz clic en "Ejecutar Recursión" para ver la pila de llamadas
📋 Código PSeInt 
Proceso CalcularFactorial
    Definir n, resultado Como Entero
    
    Escribir "Ingrese un número:"
    Leer n
    
    resultado <- Factorial(n)
    Escribir "El factorial es:", resultado
FinProceso

Funcion f <- Factorial(num)
    Definir f Como Entero
    Si num <= 1 Entonces
        f <- 1  // Caso base
    SiNo
        f <- num * Factorial(num - 1)  // Llamada recursiva
    FinSi
FinFuncion
